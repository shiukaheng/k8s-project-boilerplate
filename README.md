This is a demo project that demonstrate a monorepo structure that handles frontend, backend, surrounding infrastructure, packaging for k8s, quick iteration, and readiness for prod deployment.

Read FILE_STRUCTURE.md regarding the structure of this repo.

This repo is designed to be a super minimal skeleton that demonstrates the functionality of this structure's whole workflow, to first let the user understand how this repo is structured out.

Then, once the user understands this repo, it also is meant to be used as a starting point to build the actual app - this repo is a solid skeleton, but the user can put flesh in - iterating at lightspeed for fullstack projects, flexible enough for any microservices based project, which can then be easily scalable, and deployed anywhere thanks to the ubiquity of k8s.

The commit history shows how this repo is layered one by one.
